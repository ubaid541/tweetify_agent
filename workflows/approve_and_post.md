---
description: How to review, approve, and copy tweet drafts for manual posting
---

## Objective
Open the interactive draft review UI, approve the best tweets, and copy them for manual posting on X/Twitter.

## Required Inputs
- `.tmp/drafts_YYYY-MM-DD.json` — generated by `generate_tweets.py`

## Steps

### 1. Launch the Approval UI
```
python tools/approve_tweets.py
```
To review drafts for a specific past date:
```
python tools/approve_tweets.py --date 2026-03-10
```

---

### 2. Review Each Draft

For each tweet draft, the UI shows:
- The full tweet text
- Character count (green = safe, yellow = close, red = over)
- Source newsletter and story title
- `[THREAD]` label if the story is multi-tweet

**Your options for each draft:**

| Key | Action |
|-----|--------|
| `A` | **Approve** — mark as approved and ready to post |
| `E` | **Edit** — retype the tweet, then auto-approve |
| `S` | **Skip** — leave it out (marked as skipped) |
| `Q` | **Quit** — stop reviewing (progress is saved) |

---

### 3. After Review

At the end of the session, the UI shows:
- A summary table with all draft statuses
- Your **approved tweets printed in full** — ready to copy-paste

**Copy and paste approved tweets into X/Twitter to post manually.**

---

### 4. Re-running the Approval UI
You can exit and re-launch at any time. Only `pending` drafts will be shown in the review loop. Already-reviewed drafts appear only in the summary.

To reset a draft back to pending, manually edit `.tmp/drafts_YYYY-MM-DD.json` and set `"status": "pending"`.

---

## Notes
- **No automatic posting** — all tweets are posted manually by you
- Approved tweet statuses are saved to the JSON file with a timestamp
- Edited tweets are flagged with `"edited": true` in the file for reference
- For threads: the UI shows each tweet in the thread separately with character counts

## Error Handling
| Error | Fix |
|-------|-----|
| `drafts_YYYY-MM-DD.json not found` | Run `python tools/generate_tweets.py` first |
| `All drafts already reviewed` | All done! Summary is shown with approved tweets |
| UI not rendering colors | Ensure your terminal supports ANSI colors (Windows Terminal recommended) |
